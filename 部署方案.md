# 🚀 正式部署指南

## 🌍 雲端部署方案

### 方案一：完全雲端部署 (推薦)

#### 前端部署 - Vercel

1. 將代碼上傳到 GitHub
2. 連接 Vercel 到 GitHub 儲存庫
3. 設定環境變數：
   ```
   VITE_API_URL=https://your-backend-domain.com/api
   ```

#### 後端部署 - Railway

1. 連接 Railway 到 GitHub
2. 選擇後端資料夾
3. 自動部署並獲得網址

#### 資料庫 - MongoDB Atlas (免費)

1. 註冊 MongoDB Atlas
2. 建立免費叢集
3. 更新後端連接字符串

### 方案二：本地伺服器 + 公網存取

#### 使用 ngrok (快速測試)

```bash
# 安裝
npm install -g ngrok

# 為前端建立隧道
ngrok http 5175

# 為後端建立隧道
ngrok http 3005
```

#### 自建伺服器

- 購買 VPS (Digital Ocean, Linode 等)
- 安裝 Node.js 和 PM2
- 設定域名和 SSL

### 方案三：企業內部部署

#### 公司內部伺服器

- 在公司架設專用電腦/伺服器
- 設定固定內網 IP
- 所有員工連公司網路即可使用

## 💰 成本估算

### 免費方案

- **前端**: Vercel (免費)
- **後端**: Railway (免費額度)
- **資料庫**: MongoDB Atlas (免費 512MB)
- **總成本**: $0/月

### 付費方案

- **前端**: Vercel Pro ($20/月)
- **後端**: Railway ($5-20/月)
- **資料庫**: MongoDB Atlas ($9/月)
- **總成本**: $34-49/月

### 企業方案

- **伺服器**: $50-200/月
- **域名**: $10-15/年
- **SSL證書**: 免費 (Let's Encrypt)
- **總成本**: $50-200/月

## 🛠️ 部署準備

### 修改程式碼以支援雲端部署

1. 環境變數配置
2. 資料庫連接改為雲端
3. API 路徑動態配置
4. HTTPS 支援

## ⚡ 快速開始

### 最簡單的方案（推薦新手）

1. 使用 **ngrok** 做臨時公網存取
2. 之後升級到 **Vercel + Railway** 正式部署
3. 如果使用量大再考慮企業方案
