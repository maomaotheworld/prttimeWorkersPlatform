# 工讀生管理系統 - 資料庫遷移指南

## 問題說明

您遇到的資料丟失問題是因為原系統使用 JSON 檔案儲存資料，而 Railway 等雲端服務的檔案系統是暫時性的。每次容器重啟時，所有寫入的檔案都會被清除，只保留程式碼。

## 解決方案

將資料儲存從 JSON 檔案遷移到 PostgreSQL 資料庫，確保資料持久化。

## 設置步驟

### 1. Railway 上設置 PostgreSQL

1. 登入 Railway (https://railway.app)
2. 進入您的專案
3. 點擊 "New Service" → "Database" → "PostgreSQL"
4. Railway 會自動生成資料庫並提供連接字串

### 2. 設置環境變數

在 Railway 的環境變數中添加：

- `DATABASE_URL`: (Railway 會自動生成)
- `JWT_SECRET`: 一個安全的密鑰字串 (例如: `super-secret-jwt-key-2024`)
- `NODE_ENV`: `production`

### 3. 部署新版本

我已經修改了後端程式碼來使用 PostgreSQL：

- ✅ 添加了 `pg` 套件支援 PostgreSQL
- ✅ 創建了 `database.js` 資料庫配置檔
- ✅ 創建了 `dao.js` 資料存取層
- ✅ 修改了伺服器啟動流程

### 4. 資料庫自動初始化

新版本會自動：

- 創建所需的資料表
- 設置預設管理員帳戶 (用戶名: evelyn, 密碼: evelyn123)
- 創建預設組別

## 資料表結構

系統會自動創建以下資料表：

- `users`: 使用者帳戶
- `workers`: 工讀生資料
- `groups`: 組別資料
- `time_records`: 工時記錄
- `activity_logs`: 活動日誌

## 好處

1. **資料持久化**: 資料永久保存，不會因為容器重啟而丟失
2. **高可用性**: PostgreSQL 提供備份和復原機制
3. **效能提升**: 資料庫查詢比檔案讀寫更高效
4. **擴展性**: 支援更多並發用戶和更大資料量
5. **資料完整性**: 外鍵約束和資料驗證

## 部署後確認

1. 檢查 Railway 日誌是否顯示 "資料庫初始化完成"
2. 嘗試登入系統 (用戶名: evelyn, 密碼: evelyn123)
3. 確認可以新增和編輯工讀生資料
4. 重啟應用程式後確認資料仍然存在

## 注意事項

- 第一次部署時會自動創建資料表和預設資料
- 如果需要遷移舊資料，可以手動重新輸入或使用 Excel 匯入功能
- 建議定期備份資料庫 (Railway 提供自動備份)
