# 🚀 伺服器啟動配置

## 📋 重要端口配置

### 前端 (Frontend)

- **端口**: `5175` (固定，不可變更)
- **網址**: http://localhost:5175
- **配置**: `frontend/vite.config.js` 中設定 `strictPort: true`

### 後端 (Backend)

- **端口**: `3005`
- **API 基礎路徑**: http://localhost:3005/api

## 🔧 啟動步驟

### 1. 啟動後端

```bash
cd backend
node index.js
```

**確認訊息**: `服務器運行在 http://localhost:3005`

### 2. 啟動前端

```bash
cd frontend
npm run dev
```

**確認訊息**: `Local: http://localhost:5175/`

## ⚠️ 重要提醒

1. **前端必須運行在 PORT 5175**
   - 如果 5175 被占用，請先關閉占用該端口的程序
   - 使用 `netstat -ano | findstr :5175` 查看端口占用
   - 使用 `taskkill /PID <PID> /F` 關閉占用程序

2. **API 代理設定**
   - 前端會自動將 `/api/*` 請求代理到 `http://localhost:3005`
   - 確保後端在啟動前端之前已經運行

3. **常見問題解決**
   - 如果前端無法啟動在 5175，檢查 `vite.config.js` 中的 `strictPort: true` 設定
   - 如果 API 請求失敗，確認後端是否正常運行在 3005 端口

## 📱 打卡系統 API 端點

### 主要功能 API

- `GET /api/workers` - 獲取工讀生列表
- `GET /api/groups` - 獲取組別列表
- `GET /api/time-records` - 獲取打卡記錄
- `POST /api/time-records/clock-in` - 上班打卡
- `POST /api/time-records/clock-out` - 下班打卡
- `POST /api/time-records/edit-time` - 編輯打卡時間

### 測試 API 連接

```bash
curl http://localhost:3005/api/workers
```
